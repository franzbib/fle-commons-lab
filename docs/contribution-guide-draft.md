# Guide De Contribution Brouillon

Ce document imagine comment un enseignant pourrait contribuer plus tard. Aucun workflow de contribution n’est implémenté en v0.

Voir aussi:

- `docs/contributor-roles.md` pour les statuts futurs de personnes, ressources et contributions.
- `docs/contribution-space-concept.md` pour l'espace de contribution conceptuel.

Important: aucun compte, role reel, permission ou workflow persistant n'existe dans la v0.

## Proposer Une Ressource

Une proposition devrait contenir:

- titre;
- niveau;
- durée;
- compétence principale;
- type;
- objectifs;
- consignes;
- guide professeur si nécessaire;
- corrigé ou critères;
- licence;
- sources.

## Améliorer Une Ressource

Une amélioration peut porter sur:

- consigne plus claire;
- objectif mieux formulé;
- corrigé corrigé ou enrichi;
- variante pour un autre niveau;
- adaptation à un autre public;
- précision de durée.

## Signaler Une Erreur

Un signalement doit indiquer:

- ressource concernée;
- type d’erreur;
- passage concerné;
- proposition de correction si possible.

## Ajouter Un Retour D’Usage

Un retour d’usage utile précise:

- contexte;
- niveau réel du groupe;
- durée réelle;
- ce qui a fonctionné;
- difficultés;
- adaptations proposées.

## Proposer Une Variante

Une variante doit expliquer:

- pour quel niveau ou public;
- ce qui change;
- ce qui reste identique;
- quelles consignes adapter.

Une variante numérique ou IA doit préciser les outils possibles, les données à ne pas fournir, les risques, les garde-fous et une alternative sans IA quand c’est possible.

## Vérifier Les Droits

Avant publication publique:

- vérifier que le contenu est original ou autorisé;
- identifier les sources;
- éviter les extraits longs protégés;
- choisir une licence claire.

## Respecter Une Structure Commune

La structure recommandée reste:

- Objectifs;
- Déroulé;
- Support étudiant;
- Guide professeur;
- Corrigé;
- Variantes;
- Retours d’usage;
- Historique des versions.
