# Audit editorial des textes visibles

Date: 2026-05-23  
Outil principal: Codex

## Pages auditees

- Accueil
- A propos / Vision
- Documentation
- Technologies et IA
- Format ressource / Markdown
- Communaute
- Espace de contribution
- Etat du projet

## Contradictions ou risques reperes

- Le README indiquait encore 5 exemples Markdown alors que le validateur en verifie 8.
- La page Documentation listait les fichiers sans les organiser par usage.
- La page Format ne mentionnait pas assez clairement le role du validateur local.
- Les pages visibles devaient mieux distinguer IA documentee et appel IA reel.
- La soutenabilite future pouvait etre comprise comme un acces payant actif si elle n'etait pas recadree.
- Le prototype de depot brut devait rester clairement presente comme une maquette non connectee.

## Textes actualises

- Accueil: clarification de la v0 comme demo statique locale, sans compte, paiement, backend, Supabase ou appel IA.
- A propos: distinction plus nette entre banque de fiches et commun pedagogique.
- Technologies et IA: distinction entre ressource qui documente un usage IA et plateforme qui appellerait une API.
- Documentation: regroupement par blocs de lecture.
- Format ressource: mention du validateur, de ses limites et de la relecture enseignante.
- Communaute et Etat du projet: rappel des elements simules, conceptuels ou absents.

## Regle de maintien

Chaque page visible doit permettre de comprendre:

- ce qui fonctionne vraiment;
- ce qui est simule;
- ce qui est conceptuel;
- ce qui reste dans la roadmap future;
- ce qui est volontairement absent.
