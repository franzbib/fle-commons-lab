# Premier cycle source-mère → fiche Laforge : pronoms COD/COI

## Date

24 mai 2026

## Contexte

Un support Word réel d’enseignant FLE, contextualisé à l’ISPA à Amiens, a été utilisé comme premier cas bêta du parcours “source-mère → fiche Laforge”.

Ce support portait sur les pronoms personnels dans un thème d’enquête, autour d’un scénario de parapluie disparu. Il contenait plusieurs blocs pédagogiques : pronoms COD, pronoms COI, distinction COD/COI, `y/en`, pronoms toniques, infinitif, impératif, doubles pronoms et pronom neutre `le`.

La décision produit a été de ne pas transformer tout le document en une fiche unique, mais de le traiter comme une source-mère.

## Résultat produit

Ressource créée :

`examples/resources-markdown/grammaire-b1-plus-pronoms-cod-coi-enquete.md`

Cette ressource constitue une première extraction normalisée centrée uniquement sur les pronoms COD et COI.

## Cycle réalisé

Le cycle suivant a été mené :

1. dépôt ou prise en compte du document-source ;
2. identification du document comme source-mère riche ;
3. détection de plusieurs ressources possibles ;
4. choix d’une première extraction raisonnable : COD/COI ;
5. création d’une fiche Markdown normalisée ;
6. validation par `npm run validate:resources` ;
7. revue pédagogique prudente ;
8. passe légère de correction ;
9. validation finale par `npm run validate:resources` et `npm run build`.

## Ce que ce cycle prouve

Ce premier cycle montre que Laforge peut :

- accueillir un support enseignant réel ;
- ne pas exiger une fiche parfaite au départ ;
- valoriser le travail de l’enseignant ;
- découper une source riche en ressources possibles ;
- produire une première fiche partageable avec prudence ;
- conserver les autres blocs pour des développements futurs.

Il confirme aussi que la normalisation progressive est plus réaliste qu’une publication directe du document brut.

## Ajustements réalisés après revue

Après revue pédagogique, plusieurs ajustements ciblés ont été effectués :

- récit simplifié ;
- ajout de verbes utiles ;
- précision de durée : version courte 60 minutes, version complète 90 minutes ;
- justifications ciblées dans le corrigé ;
- variante générique renforcée avec variables ;
- clarification du statut pédagogique ;
- licence reformulée avec prudence.

## Limites assumées

La ressource reste volontairement limitée :

- elle est encore en normalisation ;
- elle n’est pas validée comme fiche étalon ;
- la licence doit être confirmée avant publication publique ;
- le traitement est limité au COD/COI ;
- les autres blocs ne sont pas traités : `y/en`, pronoms toniques, infinitif, impératif, doubles pronoms, pronom neutre `le`.

Ces limites ne sont pas des échecs. Elles permettent de garder une première fiche claire, exploitable et améliorable.

## Enseignement principal

“Un document-source enseignant ne doit pas être publié tel quel ni transformé en bloc unique. Il peut devenir une source-mère dont Laforge extrait progressivement des ressources plus courtes, plus lisibles et plus réutilisables.”

## Prochaine étape raisonnable

La prochaine étape raisonnable est :

- relire la fiche côté enseignant utilisateur ;
- éventuellement la tester en classe ;
- seulement ensuite extraire une deuxième fiche de la même source-mère.

La deuxième extraction ne doit pas être lancée avant d’avoir tiré quelques enseignements de l’usage de la fiche COD/COI.
